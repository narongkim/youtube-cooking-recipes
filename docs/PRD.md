## 1. 프로젝트 개요

### 1.1 배경 및 목적

유튜브에는 방대한 양의 요리 영상이 존재하지만, 원하는 레시피를 나중에 다시 찾거나 재료·절차를 한눈에 파악하기 어렵다. 본 서비스는 유튜브 링크 하나로 레시피 정보를 자동 추출하고, Notion 데이터베이스에 체계적으로 저장·관리할 수 있는 웹 서비스다.

Notion을 CMS로 선택한 이유는 Notion API를 통해 비개발자도 별도 관리 도구 없이 콘텐츠를 직접 편집·관리할 수 있기 때문이다.

### 1.2 프로젝트 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 유튜브 요리 레시피 관리 |
| 서비스 유형 | 웹 애플리케이션 |
| CMS | Notion API |
| 타겟 사용자 | 요리 영상을 즐겨 보며 레시피를 보관하고 싶은 개인 사용자 |

## 2. 목표 및 성공 지표

### 2.1 목표

- 유튜브 링크 입력만으로 레시피 정보(재료, 조리 절차)를 자동 추출하여 저장 시간을 최소화
- Notion을 통해 비개발자도 저장된 레시피를 자유롭게 열람·편집 가능
- 직관적인 UI로 레시피 검색·필터링 및 상세 조회 제공

### 2.2 성공 지표 (MVP 기준)

| 지표 | 목표값 |
|------|--------|
| 유튜브 링크 → 레시피 추출 성공률 | ≥ 80% |
| 추출 완료까지 소요 시간 | ≤ 30초 |
| Notion 저장 성공률 | ≥ 99% |
| 레시피 목록 첫 로딩 시간 | ≤ 2초 |

## 3. 기술 스택

### 3.1 구성

| 영역 | 기술 |
|------|------|
| Frontend | Next.js 15, TypeScript |
| CMS | Notion API |
| 스타일링 | Tailwind CSS, shadcn/ui |

| 영상 메타데이터 | YouTube oEmbed API, youtube-dl |
| 레시피 구조화 | LLM (자막/설명 → 재료·절차 파싱) |

### 3.2 아키텍처 개요

```
사용자 브라우저
    │
    ▼
Next.js 15 (App Router)
    ├── /api/extract   ← 유튜브 메타데이터·자막 추출 + LLM 구조화
    ├── /api/recipes   ← Notion DB CRUD
    └── UI 컴포넌트 (shadcn/ui)
            │
    ┌───────┴───────┐
    ▼               ▼
Notion API      YouTube API
(데이터 저장)    (영상 정보 수집)
```

## 4. Notion 데이터베이스 스키마

### 4.1 레시피(Recipe) 데이터베이스

| 필드명 (한국어) | 필드명 (영문 키) | 타입 | 설명 |
|----------------|----------------|------|------|
| 제목 | Title | Title | 레시피 이름 (예: "김치찌개") |
| 유튜브 링크 | YouTube URL | URL | 원본 영상 주소 |
| 채널명 | Channel | Text | 영상 업로더/채널 이름 |
| 썸네일 | Thumbnail | Files & media | 영상 썸네일 이미지 |
| 카테고리 | Category | Select | 한식 / 양식 / 중식 / 일식 / 기타 |
| 재료 | Ingredients | Text | 추출된 재료 목록 (마크다운) |
| 조리 절차 | Steps | Text | 추출된 조리 순서 (마크다운) |
| 즐겨찾기 | Favorite | Checkbox | 즐겨찾기 여부 |
| 상태 | Status | Select | 저장됨 / 만들어봄 / 보관함 |
| 추출 일시 | Created At | Created time | 레코드 생성 시각 (자동) |

### 4.2 카테고리 선택 옵션

- 한식
- 양식
- 중식
- 일식
- 기타

### 4.3 상태(Status) 선택 옵션

| 값 | 의미 |
|----|------|
| 저장됨 | 레시피를 추출·저장한 기본 상태 |
| 만들어봄 | 직접 요리해 본 레시피 |
| 보관함 | 나중에 만들어 볼 레시피 |

## 5. 주요 기능 명세

### 5.1 유튜브 레시피 자동 추출 및 저장

**사용자 시나리오**

1. 사용자가 유튜브 영상 URL을 입력 폼에 붙여넣기
2. 서버가 oEmbed / youtube-dl로 메타데이터(제목, 채널, 썸네일) 수집
3. 영상 자막 또는 설명문을 LLM에 전달하여 재료·조리 절차 구조화
4. 추출 결과를 미리보기로 화면에 표시
5. 사용자가 내용 확인 후 "저장" 클릭 → Notion DB에 레코드 생성

**예외 처리**

| 상황 | 처리 방식 |
|------|----------|
| 자막 없는 영상 | 영상 설명(description)으로 추출 시도, 실패 시 안내 메시지 표시 |
| 비공개/삭제 영상 | 오류 메시지 표시 및 링크 재입력 유도 |
| LLM 추출 실패 | 수동 입력 폼으로 fallback |
| 중복 URL | 기존 레시피 링크 안내 및 덮어쓰기 여부 확인 |

### 5.2 레시피 목록 조회

- Notion DB에서 레시피 목록을 가져와 카드 형태로 표시
- 각 카드: 썸네일, 제목, 채널명, 카테고리, 즐겨찾기 아이콘
- 정렬: 최신순(기본), 즐겨찾기 우선
- 페이지네이션 또는 무한 스크롤 (MVP: 페이지네이션)

### 5.3 레시피 상세 보기

- 레시피 썸네일 및 원본 영상 임베드 플레이어
- 재료 목록 (체크리스트 형태)
- 조리 절차 (번호 목록 형태)
- 즐겨찾기 토글 버튼
- 상태(Status) 변경 드롭다운
- "Notion에서 열기" 외부 링크 버튼

### 5.4 검색 및 카테고리 필터

- 검색: 제목·채널명 기준 텍스트 검색
- 필터: 카테고리(한식/양식/중식/일식/기타), 즐겨찾기, 상태
- MVP 범위: 검색 + 카테고리 필터 (날짜·복합 필터는 후순위)

## 8. 구현 단계 (로드맵)

### 단계 1: Notion 데이터베이스 설계 및 API 연동

**목표**: Notion DB 스키마 생성 및 읽기/쓰기 검증

- Notion 통합(Integration) 생성 및 토큰 발급
- 레시피 데이터베이스 스키마 생성 (섹션 4 기준)
- Next.js API Route에서 Notion SDK로 읽기/쓰기 테스트
- 환경 변수 설정 (`NOTION_TOKEN`, `NOTION_DATABASE_ID`)

**완료 기준**: Notion DB에 테스트 레코드 생성·조회 성공

---

### 단계 2: 유튜브 영상 정보 추출 파이프라인 구축

**목표**: URL 입력 → 구조화된 레시피 데이터 반환

- YouTube oEmbed API로 제목·채널·썸네일 수집
- youtube-dl(또는 yt-dlp)로 자막·설명 텍스트 수집
- LLM API 연동 → 재료 목록 및 조리 절차 JSON 파싱
- `/api/extract` 엔드포인트 구현 및 오류 처리

**완료 기준**: 다양한 요리 영상 URL 5개 이상에서 재료·절차 추출 성공

---

### 단계 3: Next.js UI 구현 및 저장 연동

**목표**: 목록/추가/상세 화면 완성 및 전체 흐름 동작

- shadcn/ui 컴포넌트로 홈·추가·상세 화면 구현
- 추출 결과 미리보기 → 저장 버튼 → Notion 저장 흐름 완성
- 검색·카테고리 필터 UI 및 Notion API 필터 쿼리 연동
- 즐겨찾기·상태 토글 → Notion 레코드 업데이트 연동

**완료 기준**: 유튜브 URL 입력부터 상세 조회까지 전체 흐름 E2E 동작 확인

## 9. 제약 사항 및 리스크

| 항목 | 내용 | 대응 방안 |
|------|------|----------|
| YouTube 자막 없는 영상 | 자막 미제공 영상은 설명 텍스트만 사용 가능 | 설명 → LLM 추출 시도 후 수동 입력 fallback |
| Notion API 속도 제한 | 초당 3회 요청 제한 | 요청 큐잉 및 재시도 로직 구현 |
| LLM 비용 | 영상별 자막 토큰 소모 | 자막 전처리로 입력 길이 최소화 |
| youtube-dl 정책 변경 | YouTube 구조 변경 시 자막 추출 실패 가능 | yt-dlp 유지보수 버전 사용, oEmbed 병행 |

## 10. 용어 정리

| 용어 | 설명 |
|------|------|
| CMS | Content Management System. 콘텐츠 저장·관리 시스템 |
| Notion Integration | Notion API 접근을 위한 서버 측 인증 토큰 |
| oEmbed | URL만으로 미디어 임베드 정보를 반환하는 표준 API |
| yt-dlp | youtube-dl의 포크 프로젝트. 자막·메타데이터 추출에 사용 |
| LLM | Large Language Model. 자막 텍스트를 재료·절차로 구조화하는 데 활용 |
